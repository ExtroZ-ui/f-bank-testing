# Итоговое задание по дисциплине «Тестирование прикладного ПО»

## 📌 Описание проекта

В рамках итогового задания выполнено тестирование веб-сервиса перевода денежных средств с банковского счёта на банковскую карту.

В процессе работы были выполнены:

- ручное тестирование приложения;
- разработка тестовой документации;
- составление баг-репортов;
- автоматизация тестирования с использованием Selenium + Python + pytest;
- настройка CI через GitHub Actions.

---

## 📂 Структура репозитория

```text
manual/
├── test-design.md       — тест-дизайн
├── test-cases.md        — ручные тест-кейсы
└── bug-reports.md       — баг-репорты

autotests/
├── requirements.txt     — зависимости Python
└── tests/
    └── test_bank_ui.py  — Selenium автотесты

.github/workflows/
└── test.yml             — CI (GitHub Actions)

dist/                    — тестируемое приложение
```

---

## 🧪 Ручное тестирование

В ходе ручного тестирования были проверены следующие сценарии:

- успешный перевод средств;
- отказ при недостатке средств;
- корректность расчёта комиссии;
- проверка граничных значений;
- валидация номера карты;
- валидация суммы перевода;
- проверка пользовательского интерфейса.

---

## 🐞 Найденные дефекты

В результате тестирования были выявлены следующие дефекты:

| ID | Описание |
|---|---|
| BR-01 | Некорректный расчёт комиссии за перевод |
| BR-02 | Принимается номер карты длиной более 16 цифр |
| BR-03 | Разрешён ввод отрицательной суммы перевода |
| BR-04 | Кнопка перевода отображается без ввода суммы |

Для каждого дефекта:
- оформлен отдельный баг-репорт;
- реализован отдельный автоматизированный Selenium-тест.

---

## 🤖 Автоматизированное тестирование

Автотесты реализованы с использованием:

- Python 3;
- pytest;
- Selenium WebDriver;
- Google Chrome Headless.

Реализованы проверки следующих сценариев:

### Позитивные сценарии
- успешный перевод;
- отказ при недостатке средств.

### Негативные сценарии
- некорректный расчёт комиссии;
- номер карты более 16 цифр;
- отрицательная сумма перевода;
- перевод без указания суммы.

---

## ⚙️ Локальный запуск проекта

### Требования

Для запуска проекта необходимо установить:

- Python 3.11+
- Google Chrome
- ChromeDriver (совместимый с версией Chrome)

---

## ▶️ Запуск тестируемого сервиса

Перейти в директорию с приложением:

```bash
cd dist
```

Запустить локальный HTTP-сервер:

```bash
python3 -m http.server 8000
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:8000/?balance=30000&reserved=20001
```

---

## 🧠 Установка зависимостей

Из корня проекта:

```bash
pip install -r autotests/requirements.txt
```

---

## ▶️ Запуск автотестов

Запуск всех тестов:

```bash
python -m pytest -v autotests/tests
```

---

## 📊 Ожидаемый результат выполнения тестов

Часть тестов должна завершаться с ошибкой.

Это ожидаемое поведение, так как автотесты воспроизводят найденные дефекты приложения.

Пример результата:

```text
2 passed
4 failed
```

Падающие тесты соответствуют следующим дефектам:

- BR-01 — некорректный расчёт комиссии;
- BR-02 — некорректная валидация номера карты;
- BR-03 — отсутствует проверка отрицательной суммы;
- BR-04 — кнопка перевода отображается без суммы.

---

## 🔁 CI (GitHub Actions)

Настроен автоматический запуск тестов при:

- push;
- pull request.

CI выполняет:

1. установку зависимостей;
2. запуск тестируемого приложения;
3. запуск Selenium-тестов;
4. проверку найденных дефектов.

---

## ⚠️ Важно

Итоговая сборка GitHub Actions является `FAILED` намеренно, требование задания.
