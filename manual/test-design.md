# Тест-дизайн ручного тестирования

## 1. Цель тестирования
Проверить корректность работы веб-сервиса перевода денежных средств:
- выбор счёта;
- ввод номера карты;
- ввод суммы перевода;
- расчёт комиссии;
- проверка доступности перевода.

---

## 2. Объект тестирования
Веб-сервис для перевода денежных средств со счёта на банковскую карту.

---

## 3. Основание для тестирования
Согласно условиям задания:

- номер карты должен состоять из **ровно 16 цифр**;
- доступная сумма рассчитывается как:

  доступная сумма = баланс − резерв

- комиссия = **10% от суммы перевода**, округляется вниз;
- перевод возможен, если сумма с комиссией **не превышает доступную сумму**;
- если превышает — отображается сообщение об ошибке.

---

## 4. Тестовые техники
При проектировании тестов использованы:

- позитивные сценарии (успешные операции);
- негативные сценарии (ошибки);
- граничные значения;
- классы эквивалентности.

---

## 5. Область покрытия

Проверены следующие сценарии:

1. Успешный перевод при достаточном остатке;
2. Отказ при недостатке средств;
3. Граничное значение (равенство суммы и доступного остатка);
4. Валидация номера карты;
5. Валидация суммы перевода.
